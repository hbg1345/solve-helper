v0.5.2 2026-03-25
PERF Google Fonts render-blocking waterfall 제거
문제
FCP/LCP 3초 이상 — Orbitron 폰트 로드가 렌더링을 차단
원인
globals.css에서 @import url()로 Google Fonts를 로드하여 HTML → CSS → Google Fonts CSS → 폰트 파일 3단계 네트워크 waterfall 발생
해결
@import 삭제, next/font/google의 Orbitron으로 교체. CSS variable(--font-orbitron)로 주입하고 tailwind.config의 font-game이 이를 참조하도록 변경
결과
Next.js가 폰트를 self-host + preload하여 render-blocking 네트워크 왕복 제거 (예상 -1~2초)

문제
모든 페이지에서 직렬 Supabase 호출로 TTFB 지연
원인
AppShell에서 cookies()→createClient() 직렬 호출, AuthButton에서 getUser() (네트워크 왕복) + 중복 user_info 쿼리
해결

결과
매 요청마다 Supabase auth 서버 네트워크 왕복 1회 제거 (예상 -0.5~1초)

v0.5.1 2026-03-19
FIX profile 페이지 Vercel 빌드 오류 (cookies() Suspense 외부 호출)
문제
Vercel 빌드 실패 — "Route /profile: Uncached data was accessed outside of <Suspense>"
원인
v0.5.0에서 다국어 지원 구현 시 ProfilePage(outer, non-Suspense)를 async로 변경하고 getServerTr() → cookies()를 호출. Next.js 16에서는 동적 API(cookies/headers)를 Suspense 경계 밖에서 호출하면 정적 렌더링 시 오류 발생
해결
ProfilePage를 다시 non-async으로 되돌리고, getServerTr() 호출을 Suspense 내부의 UserDetails() async 컴포넌트 안으로 이동. subtitle 텍스트도 UserDetails 안에서 렌더링
결과
Vercel 빌드 성공
